## 5. The vendoring contract (read once, gotcha-prone)

The package ships an immutable snapshot of the upstream `mixedmodels`
crate plus all transitive registry deps so the tarball is self-contained
(CRAN policy + R-Universe build hosts).

Pipeline lives in `tools/vendor-rust.R`. It does:

1. Copies `/Users/bbuchsbaum/code/rust/mixedmodels` (configurable via
   `MIXEFF_UPSTREAM_PATH`) → `src/rust/upstream/mixedmodels/`. Whitelist:
   `Cargo.toml`, `Cargo.lock`, `src/`, plus a synthesized `LICENSE-MIT`.
   Tests, datasets, benchmarks, the Julia source, and other large
   non-compile inputs are excluded.
2. Rewrites `src/rust/Cargo.toml` path dep to `upstream/mixedmodels`.
3. `cargo vendor` → `src/vendor/` (~69 crates, ~4 MB on disk).
4. Tars `src/vendor/` → `src/rust/vendor.tar.xz` (4.1 MB).
5. Writes `src/rust/vendor-config.toml` with `directory = "vendor"`.

Why both forms? `R CMD build`'s clean step removes `src/vendor/`, so it
must be reconstituted from `vendor.tar.xz` at `R CMD INSTALL` time.
`src/Makevars.in` honors either form; `tools/config.R` detects either to
gate the `--offline` flag.

The `directory = "vendor"` (without `../`) is intentional: cargo treats
paths in `CARGO_HOME`'s config as anchored at `CARGO_HOME`'s parent
(= the package `src/` directory at install time). `directory = "../vendor"`
canonicalizes to `<repo>/vendor` and breaks. We learned this the hard
way; do not "simplify" the path.

Re-run `Rscript tools/vendor-rust.R` whenever the upstream crate or its
`Cargo.lock` changes.

## 10. Gotchas — read these once

1. **rlang reserves `header`, `body`, `footer` as condition-field names.**
   See the comment block at the top of `R/conditions.R`. Use `input`,
   `value`, `subject`, etc. Setting a custom `header = <list>` field
   makes `cnd_header()` raise on print.
2. **The path dep on `mixedmodels` must be relative, not absolute.**
   `'upstream/mixedmodels'` from `src/rust/Cargo.toml` is the canonical
   form (relative to the manifest); absolute paths break tarball
   relocation. The `'../../../rust/mixedmodels'` form is for *dev*
   setups that haven't run `tools/vendor-rust.R` yet (it gets rewritten
   by the script).
3. **Vendor config path: `directory = "vendor"`, not `"../vendor"`** —
   see §5 above.
4. **`chmod +x` via Bash is sandbox-blocked.** Use `Sys.chmod()` from R.
   Affects `configure`, `configure.win`, `cleanup`, `cleanup.win` after
   `rextendr::use_extendr()`.
5. **`cargo clean` instead of `rm -rf src/rust/target`** — also sandbox.
6. **`devtools::test()` builds in place; `R CMD check` builds out of
   place** — both must work. The Cargo + vendor + Makevars stack is
   what makes both possible.
7. **Local-only Homebrew tools: `cmake`, `checkbashisms`.** Without
   them, R CMD check WARNINGs surface that won't appear on CRAN. They
   are not package issues.
8. **Vignette index entries must match `\VignetteIndexEntry{}` in the
   Rmd YAML preamble** — easy to typo.

## 11. If you need to extend the typed-condition catalog

Pattern (see `R/schema.R` for the canonical example):

```r
mm_my_new_error <- function() {  # the verb that detects the condition
  mm_abort(
    message = "human-readable description",
    class   = "mm_my_new_error",   # automatically gains "mm_condition"
    input   = the_offending_value,  # NOT `header`/`body`/`footer`
```

`mm_split_tagged_error()` routes it correctly. See `mm_parse_formula()`
or `mm_json_negotiate()` for the round trip.

the audit-first surface; the Rust side is the source of truth. Phase 0
